简单应用 ​

TIP

支持 JS 写法,但文档会以 TS 写法进行

进入 plugins 目录后创建文件夹

shell
cd plugins
mkdir hello-plugin

在插件目录中创建 index.ts

shell
cd hello-plugin
vi index.ts #需要学习linux vi指令

创建应用 ​

阿柠檬会识别该目录下的 index.ts 文件

并实例化 apps 目录中的所有实例文件

ts
import { createApp } from 'alemon'
/* 非依赖引用 */
import { AppName } from './app.config.js'
/** 创建实例化对象 */
const app = createApp(AppName)
/** 重定义e消息 */
app.setMessage(e => {
  // 用户编号
  e.user_id = e.user_id || e.msg?.author?.id
  // 用户头像
  e.user_avatar = e.user_avatar || e.msg?.author?.avatar
  return e
})
/** 加载apps目录 */
await app
  .create('apps')
  .then(() => {
    console.log('《柠檬测试》启动')
    return
  })
  .catch(err => {
    console.log(err)
    return
  })
/** 挂载 */
app.mount()

创建匹配 ​

apps 目录中创建 word.ts

shell
mkdir apps
cd apps
vi word.ts

此时,我们需要在alemon中引入plugin类与Messagetype消息类型

ts
import { plugin, Messagetype } from "alemon";

同时并继承plugin类为word类

ts
export class word extends plugin {
  constructor() {
    super();
  }
}

而后我们在super中传入一个对象

此对象必须包含一个rule(数组类型)属性

ts
super({
  rule: [],
});

数组中包含了多个匹配的reg正则和fnc描述方法

ts
[
  {
    reg: /^\/你好$/,
    fnc: "postHello",
  },
];

我们在指令中描述了postHello方法

所以我们还需要定义一个实际执行的方法

注意该方法必须是一个异步方法,且返回一个boolean值

返回为false表示执行后继续匹配其他可能能匹配到的方法

反之,当返回为true时,将直接结束匹配行为

ts
async postHello(e: Messagetype): Promise<boolean> {
    return false;
  }

我们机器人会传入一个e消息对象

ts
e.reply("你好");

e对象有一个reply()方法是专用于发送文本消息的

可以发送任何想要回复的字符消息

甚至是以数组形式存在的数组消息

除此之外,还有一些模板消息需要自己去了解

应该如下所示

ts
import { plugin, Messagetype } from "alemon";
export class word extends plugin {
  constructor() {
    super({
      rule: [
        {
          reg: /^\/你好$/,
          fnc: "postHello",
        },
      ],
    });
  }
  async postHello(e: Messagetype): Promise<boolean> {
    e.reply("你好");
    return false;
  }
}

TIP

这种在 e 消息对象上执行属性方法会是插件开发的常见用法
